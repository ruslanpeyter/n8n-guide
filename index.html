Create a VM Instance

Sign up for Oracle Cloud

  • 1
    Go to oracle.com/cloud/free and sign up
  • 2
    Provide a valid credit card — for identity verification only, you won't be charged
  • 3
    Choose your home region — it cannot be changed later

Create the VM

  • 1
    Go to Compute → Instances → Create Instance
  • 2
    Enter a name (e.g. n8n-server)
  • 3
    Under Shape, click Change Shape → select VM.Standard.A1.Flex (ARM) → set 4 OCPUs and 24 GB RAM
  • 4
    Under Image, select Ubuntu 22.04
  • 5
    Under SSH Keys, paste your public key or let Oracle generate one
  • 6
    Click Create
💡 Generate an SSH key pair with: ssh-keygen -t ed25519 -C "n8n-oracle" — then paste the contents of ~/.ssh/id_ed25519.pub into the SSH key field.

Connect via SSH

bash
ssh -i ~/.ssh/id_ed25519 ubuntu@YOUR_PUBLIC_IP

Open Firewall Ports

Oracle Cloud has two firewall layers — both must be configured. This is the most common reason n8n is unreachable after setup.

Layer 1 — Oracle Cloud Security List

  • 1
    Go to Networking → Virtual Cloud Networks → Your VCN
  • 2
    Click Security Lists → Default Security List → Add Ingress Rules
  • 3
    Add three rules — one for each port below:
Source CIDRProtocolPortPurpose
0.0.0.0/0TCP80HTTP (needed for SSL cert)
0.0.0.0/0TCP443HTTPS
0.0.0.0/0TCP5678n8n direct access (optional)

Layer 2 — OS Firewall (Ubuntu)

bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 5678/tcp
sudo ufw enable
sudo ufw status
⚠️ Ubuntu does not use firewall-cmd — that's for Oracle Linux / CentOS. Use ufw on Ubuntu.

Add Swap Memory

Required if you're on the 1 GB AMD instance (E2.1.Micro). Skip if you're on ARM A1.Flex with 24 GB.

bash
# Create 2GB swap file
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# Make permanent across reboots
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Tune swappiness (use RAM first)
echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# Verify
free -h

Install n8n

Step 1 — Generate encryption key

This key encrypts your saved credentials. Save it somewhere safe — losing it means losing access to all stored credentials.

bash
head /dev/urandom | tr -dc A-Za-z0-9_ | head -c 32 ; echo ''
⚠️ Copy the output and save it in a password manager before proceeding.

Step 2 — Create the docker-compose file

bash
mkdir ~/n8n && cd ~/n8n

cat > docker-compose.yml << 'EOF'
services:
  n8n:
    image: n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_ENCRYPTION_KEY=YOUR_GENERATED_KEY
      - WEBHOOK_URL=http://YOUR_PUBLIC_IP:5678
      - GENERIC_TIMEZONE=Europe/Kiev
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
EOF

Step 3 — Start n8n

bash
docker compose up -d

# Watch logs until you see "Editor is now accessible"
docker compose logs -f
✅ Once you see Editor is now accessible via: http://YOUR_IP:5678 — open that URL in your browser and create your admin account.

Nginx + HTTPS (SSL)

This gives you https://n8n.yourdomain.com instead of http://IP:5678. Required for webhooks to work with most external services.

Install Nginx and Certbot

bash
sudo apt install nginx certbot python3-certbot-nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx

Create Nginx config for n8n

bash
cat > /etc/nginx/sites-available/n8n << 'EOF'
server {
    listen 80;
    server_name n8n.yourdomain.com;

    location / {
        proxy_pass http://localhost:5678;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        chunked_transfer_encoding on;
        proxy_buffering off;
        proxy_read_timeout 3600s;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/n8n /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

Get SSL certificate

bash
sudo certbot --nginx -d n8n.yourdomain.com
💡 Certbot will ask for your email, agree to terms, then automatically configure HTTPS and set up auto-renewal. Certificates renew every 90 days automatically.

Update docker-compose.yml for HTTPS

bash
cat > ~/n8n/docker-compose.yml << 'EOF'
services:
  n8n:
    image: n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_ENCRYPTION_KEY=YOUR_KEY_HERE
      - WEBHOOK_URL=https://n8n.yourdomain.com
      - N8N_HOST=n8n.yourdomain.com
      - N8N_PROTOCOL=https
      - GENERIC_TIMEZONE=Europe/Kiev
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
EOF

cd ~/n8n && docker compose down && docker compose up -d

Backup

Manual backup

bash
docker cp n8n-n8n-1:/home/node/.n8n/database.sqlite \
  ~/n8n-backup-$(date +%Y%m%d).sqlite

# Verify
ls -lh ~/n8n-backup*.sqlite

Automatic daily backup (cron)

bash
crontab - << 'EOF'
0 2 * * * docker cp n8n-n8n-1:/home/node/.n8n/database.sqlite /home/ubuntu/n8n-backup-$(date +\%Y\%m\%d).sqlite
EOF

Download to local machine (SCP)

bash
scp -i ~/.ssh/id_ed25519 ubuntu@YOUR_IP:~/n8n-backup-*.sqlite ~/Desktop/
⚠️ Always back up before updating or rebooting. The SQLite file contains everything — workflows, credentials, executions, users.

Update n8n (Correct Sequence)

Always follow this exact sequence. Skipping docker compose down causes a port already allocated error because the old container keeps holding port 5678.

bash
# 1. Backup first
docker cp n8n-n8n-1:/home/node/.n8n/database.sqlite ~/n8n-backup-$(date +%Y%m%d).sqlite

# 2. Pull latest image
cd ~/n8n
docker compose pull

# 3. Stop old container
docker compose down

# 4. Start with new image
docker compose up -d

# 5. Clean up old images (after new one is running)
docker image prune -f

# 6. Verify
docker compose logs -f | grep Version

Auto-Update Script

Create the script

bash
cat > ~/n8n/update.sh << 'EOF'
#!/bin/bash
docker cp n8n-n8n-1:/home/node/.n8n/database.sqlite \
  /home/ubuntu/n8n-backup-$(date +%Y%m%d).sqlite
cd /home/ubuntu/n8n
docker compose pull
docker compose down
docker compose up -d
docker image prune -f
ls -t /home/ubuntu/n8n-backup-*.sqlite | tail -n +8 | xargs rm -f
EOF

chmod +x ~/n8n/update.sh

Schedule with cron (every Sunday 3:00 AM)

bash
crontab - << 'EOF'
0 3 * * 0 /home/ubuntu/n8n/update.sh >> /home/ubuntu/n8n-update.log 2>&1
EOF

Check update log

bash
cat ~/n8n-update.log

Workflow Export (JSON)

Export workflows as JSON for portability — useful for sharing, version control, or restoring to a new instance.

Export all workflows

  • 1
    Open your n8n instance in the browser
  • 2
    Go to Settings → Import/Export
  • 3
    Click Export all workflows — downloads a .json file
  • 4
    Store it in Google Drive, GitHub, or locally
Backup typeContains
SQLite fileEverything: workflows, credentials, executions, users, settings
JSON exportWorkflows only — no credentials or execution history
⚠️ JSON export does NOT include credentials — you'll need to re-enter API keys if restoring to a fresh instance.

Troubleshooting

ProblemFix
n8n not accessibleCheck docker ps + both Oracle Security List AND ufw (port 80/443/5678)
Port already allocateddocker stop $(docker ps -q) && docker rm $(docker ps -aq), then docker compose up -d
Out of memoryfree -h to check. If swap inactive: sudo swapon /swapfile
Nginx not workingsudo nginx -t then sudo systemctl restart nginx
SSL expiredsudo certbot renew
firewall-cmd not foundUse sudo ufw instead — Ubuntu does not use firewall-cmd
n8n not startingdocker compose logs -f to read the error
Disk fulldocker system prune -f then df -h
Webhooks show wrong URLUpdate WEBHOOK_URL and N8N_HOST in docker-compose.yml, restart

Full system check (run anytime)

bash
docker ps                        # containers running?
free -h                          # memory ok?
df -h                            # disk space?
sudo systemctl status nginx      # nginx running?
sudo certbot certificates        # SSL valid?
docker system df                 # docker disk usage
